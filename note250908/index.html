    作用域链

    如果发生了函数嵌套的情况：

    function f() {
    var num = 10;
    function fn() {
        console.log(num);
    }

}

    内部函数可以访问外部函数的变量，用链式查找决定哪些数据能被内部函数访问，称为作用域链。

    预解析

    • JS运行时分为两步：预解析、代码执行。

    • 预解析时，JS引擎会把代码里所有的var和function都提升到当前作用域的最前面。

    • 执行时按照代码书写的顺序从上到下执行。

    • 变量预解析/变量提升：把所有变量声明提升到作用域最上面（不提升赋值操作）

    • 函数预解析/提升：把所有函数声明提升到作用域最上面（不调用函数）

    箭头函数 => (来自AI)

    箭头函数（=>）是 ES6 新增的函数语法，与普通函数（function 声明）在语法和行为上有显著区别，核心差异体现在 this 指向、语法简洁性、使用场景 等方面。下面详细对比：

    1. 语法差异

    普通函数

    • 必须用 function 关键字声明，可包含完整的函数体（{ } 和 return）

    javascript // 函数声明 function add(a, b) { return a + b; } // 函数表达式 const multiply = function(a, b) { return a * b; };sd

    箭头函数

    • 用 => 替代 function 关键字，语法更简洁

    • 单参数可省略括号 ()，单语句返回可省略 { } 和 return（隐式返回）

    javascript // 基础形式（多参数） const add = (a, b) => { return a + b; }; // 简化：单语句返回可省略 { } 和 return const add = (a, b) => a + b; // 简化：单参数可省略 () const double = num => num * 2; // 特殊：返回对象时需用 () 包裹（避免 { } 被解析为代码块） const getUser = () => ({ name: "张三", age: 20 });

    2. 核心差异：this 指向

    这是箭头函数与普通函数最关键的区别，直接影响它们的使用场景。

    普通函数

    • this 指向 调用时的上下文对象，取决于“谁调用它”：

    • 全局调用时，this 指向全局对象（浏览器中是 window，Node.js 中是 global）

    • 对象方法调用时，this 指向该对象

    • 事件绑定中，this 指向触发事件的元素

    • 用 call/apply/bind 可修改 this 指向

      javascript const obj = { name: "张三", sayHi: function() { console.log(this.name); // "张三"（this 指向 obj） } }; obj.sayHi(); // 全局调用时 this 指向 window const globalFunc = obj.sayHi; globalFunc(); // undefined（浏览器中 window.name 默认为空）

    箭头函数

    • 没有自己的 this，this 继承自 声明时的父级作用域（“词法作用域 this”）

    • 无法通过 call/apply/bind 修改 this 指向（参数会被忽略）

    javascript const obj = { name: "张三", sayHi: () => { console.log(this.name); // undefined（this 继承自父级，即全局 window） }, delaySayHi: function() { // 普通函数中的 this 指向 obj setTimeout(() => { // 箭头函数的 this 继承自 delaySayHi 的 this（即 obj） console.log(this.name); // "张三" }, 1000); } }; obj.sayHi(); // undefined obj.delaySayHi(); // 1秒后输出 "张三"

    经典场景：箭头函数常用于回调函数（如 setTimeout、数组方法），避免普通函数中 this 指向混乱的问题。

    3. 其他差异

    特性 普通函数 箭头函数
    arguments 对象 有（包含所有参数的类数组） 无（需用剩余参数 ...args）
    构造函数 可作为构造函数（new Func()） 不可（会报错）
    原型 prototype 有原型属性 无原型属性
    Generator 函数 可使用 function* 声明 不可（不支持 yield）

    示例：arguments 与剩余参数

    // 普通函数：用 arguments 获取参数
function sum() {
let total = 0;
for (let i = 0; i < arguments.length; i++) {
total += arguments[i];
}
return total;
}
sum(1, 2, 3); // 6
// 箭头函数：用 ...args 剩余参数替代 arguments
const sum = (...args) => {
return args.reduce((total, num) => total + num, 0);
};
sum(1, 2, 3); // 6

    4. 使用场景建议

    优先用箭头函数的场景

    • 回调函数：如 setTimeout、map、filter 等，避免 this 指向问题

    javascript const numbers = [1, 2, 3]; const doubled = numbers.map(num => num * 2); // 简洁且无 this 困扰

    • 简短逻辑函数：单语句返回的简单函数（如工具函数）

    • 需要继承父级 this 的场景：如对象方法中的嵌套函数（见上文 delaySayHi 示例）

    优先用普通函数的场景

    • 对象方法：因为箭头函数的 this 不指向当前对象

    javascript const obj = { value: 10, getValue: function() { // 必须用普通函数 return this.value; } };

    • 构造函数：需要通过 new 创建实例时

    javascript function User(name) { this.name = name; // 普通函数可作为构造函数 } const user = new User("张三");

    • 需要 arguments 对象或 prototype 时

    总结

    箭头函数的设计初衷是 简化语法 并 解决普通函数中 this 指向混乱 的问题，但它并非普通函数的替代品。实际开发中需根据场景选择：需要动态 this 时用普通函数，需要简洁语法或固定 this 时用箭头函数。理解 this 指向的差异是掌握两者的关键。
