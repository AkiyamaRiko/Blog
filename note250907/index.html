    arguments

    当我们不确定有多少个参数的时候，可以用arguments来获取。arguments实际上是所有函数都有的一个内置对象（也只有函数才有这个对象），存储了所有传递过来的实参。

    arguments的展示形式是伪数组，具有以下特点：

    • 具有length属性

    • 按索引存储数据

    • 不具有push、pop等方法

    例：找出任意个数的最大值。

    function getMax() {
    var max = arguments[0];
    for (var i = 1; i < arguments.length; i++)
        //因为已经把max设为参数的第一个了，所以遍历的时候从第二个开始。
        if (arguments[i] >= max) {
            max = arguments[i];
        }
    return max;
}

console.log(getMax(4,54,667,293,375,29)); // 调用

    例：用函数翻转数组

    function reverse(arr) {
    var newArr = [];
    for (var i = arr.length - 1; i >= 0; i--){
        // 这里是从倒数第一个值开始往前走，所以用 i--；
        newArr[newArr.length] = arr[i];
    }
    return newArr;
}

var arr1 = reverse([1, 3, 5, 7, 9]);
console.log(arr1);

    函数的另一种声明方式：表达式法

    声明时：var 变量名 = function ( ){ 需要封装的代码 };

    调用时：变量名( );

    这种声明方式和一般声明变量的方式差不多。函数表达式也可以传递参数。

    作用域

    指的是代码名（变量）起作用的范围，分为全局作用域和局部作用域。

    • 全局作用域：整个Script标签内，或者一个单独的js文件内。

    • 局部作用域：只在函数内部起作用，也叫函数作用域。

    根据作用域不同，可以把变量分为全局变量和局部变量。

    • 注意：如果在函数内部，没有声明，直接赋值，该变量会被视为全局变量。

    let和const（来自AI）

    原本的课程里没有，所以在这里补充。

    let 和 const 是 ES6（2015年）新增的变量声明方式，用于解决传统 var 存在的作用域混乱、变量提升等问题，是现代 JavaScript 开发的基础。下面我们来详细对比它们的特性和使用场景：

    1. let：用于声明可修改的变量

    核心特性：
    • 块级作用域：只在 { } 包裹的代码块（如 if、for、函数体等）内有效，外部无法访问。

    javascript if (true) { let a = 10; console.log(a); // 10（块内可访问） } console.log(a); // 报错：a is not defined（块外不可访问）

    • 不存在变量提升：必须先声明后使用，否则会报错（var 会提升到作用域顶部，值为 undefined）。

    javascript console.log(b); // 报错：Cannot access 'b' before initialization let b = 20;

    • 不允许重复声明：同一作用域内不能重复声明同名变量（包括 var 声明的变量）。

    javascript let c = 30; let c = 40; // 报错：Identifier 'c' has already been declared var d = 50; let d = 60; // 同样报错

    • 暂存死区（TDZ）：在变量声明前的区域，变量处于“暂存死区”，无法被访问（避免了 var 的意外赋值问题）。

    #### 2. const：用于声明不可修改的常量

    ##### 核心特性：

    • 声明时必须赋值：否则会报错（let 可以先声明后赋值）。

    javascript const e; // 报错：Missing initializer in const declaration const e = 70; // 正确

    • 块级作用域：和 let 一样，只在当前代码块内有效。

    javascript { const f = 80; console.log(f); // 80 } console.log(f); // 报错：f is not defined

    • 不允许重复声明：规则和 let 一致。

    • 值不可修改？（注意！）：

    对于 基本数据类型（数字、字符串、布尔值等）：值确实不可修改，强行修改会报错。

    javascript const g = 90; g = 100; // 报错：Assignment to constant variable

    对于 引用数据类型（对象、数组等）：变量存储的是“引用地址”，地址不可改，但对象/数组的 内部属性可以修改。

    javascript const obj = { name: "张三" }; obj.name = "李四"; // 允许：修改对象内部属性 console.log(obj); // { name: "李四" } obj = { age: 20 }; // 报错：试图修改引用地址

    3. let、const 与 var 的对比

    特性 var let const
    作用域 函数级/全局 块级 块级
    变量提升 有（提升到作用域顶） 无（暂存死区） 无（暂存死区）
    重复声明 允许 不允许 不允许
    初始赋值 可选 可选 必须
    值可修改 允许 允许 基本类型不可改，引用类型内部可改

    4. 最佳实践

    • 优先使用 const：如果变量的值不需要修改（如配置项、固定常量），用 const 可以避免意外修改，代码更安全。

    • 其次使用 let：当变量需要被重新赋值时（如循环计数器、条件判断的临时变量），再用 let。

    • 避免使用 var：var 的作用域和提升特性容易导致 bugs（如循环中闭包问题），现代开发中已基本被 let/const 替代。 示例：

    javascript // 推荐写法 const PI = 3.14159; // 常量用 const let count = 0; // 会被修改的变量用 let for (let i = 0; i < 5; i++) { // 循环变量用 let（块级作用域，每次循环都是新变量） count += i; }

    掌握 let 和 const 是理解 ES6 及后续版本的基础，也是现代前端框架（如 Vue、React）开发中的必备知识。
