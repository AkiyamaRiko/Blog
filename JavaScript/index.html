    变量是存放数据的容器。

    变量的数据类型¶

    JS的数据变量类型是赋值的时候才确定的，并且类型可以变化。

    • 简单数据类型：Number（包括整数和小数）、Boolean（布尔型，如true、false）、String（字符串）、Undefined（未定义）、Null（空值）。

    • 复杂数字类型：Object等。

    编译与解释¶

    • 计算机只能理解机器语言。程序语言翻译成机器语言的工具称为翻译器。

    • 编译与解释的区别在于翻译的时间点不同。

    • 编译器是在代码执行之前编译，生成中间代码文件。例如JAVA

    • 解释器是在运行时进行即时解释，并立即执行。例如JS

    用来实现赋值、比较、执行算术运算等功能的符号。

    • 加减乘除

    • % 取余数（返回除法的余数）例如 9 % 2 = 1

    表达式和返回值¶

    • 表达式是由数字、变量、运算符组成的式子。

    • 表达式最终都会有一个结果返回，称为返回值。

    递增和递减运算¶

    用于反复给数字变量+1或-1

    // 前置自增：先加1再返回值
var num = 10;
++ num; // 类似于num = num + 1
console.log(num); // 11

// 后置自增：先返回原值再加1
var num = 10;
num ++; // 类似于num = num + 1
console.log(num); // 10
console.log(num); // 11

    比较运算¶

    比较两个数据，然后返回布尔值。（使用==时会把字符串数据转换为数字型）

    逻辑运算符¶

    • && 与(and)

    • || 或(or)

    • ！非(not)

    • && 比 || 的权重高

    console.log(3 > 5 && 3 > 2); // false
colsole.log(3 > 5 && 3 > 2); // true
console.log(!true); // false
    • 逻辑中断（短路）：当有多个表达式参与逻辑运算时，如果左边的表达式可以确定结果，就不再运算右边。
    console.log(123 && 456); // 456
console.log(0 && 456); // 0
// 表达式1 && 表达式2
// 如果表达式1为真，则返回表达式2
// 如果表达式1为假，则返回表达式1
// 从左开始，如果是真，就继续往下运算直到最后一个，返回最后一个值
// 如果是假，就返回当前的值

    console.log(123 || 456); // 123
console.log(0 || 456); //456
// 表达式1 || 表达式2
// 如果表达式1为假，则返回表达式2
// 如果表达式1为真，则返回表达式1
// 从左开始，如果是假，就继续往下运算直到最后一个，返回最后一个值
// 如果是真，就返回当前的值

    赋值运算符¶

    • = 直接赋值

    • += -= 加/减一个数以后再赋值

    • *= /= %= 乘、除、取模以后再赋值

    var num = 10;
num += 2; // 相当于num = num + 2
console.log(num); // 12

    流程控制¶

    简单来说：控制代码按照什么样的结构顺序执行

    分支流程、if语句¶

    if语句的语法结构：

    if ('条件表达式'){
    // 执行语句
}

    执行思路：如果条件表达式结果为true，则执行；如果条件表达式为false，则跳过大括号里的语句，执行if语句后面的代码。

    var age = prompt('输入你的年龄')
if (age >= 18){
    alert('你成年了')
} else {
    alert('你打游戏有防沉迷')
}

    多分支语句（多选一操作）：

    if ('条件表达式') {
    // 语句1
} else if('条件2'){
    // 语句2
} else if('条件3'){
    // 语句3
} else ('条件4'){
    // 最后的语句
}

    执行思路：如果满足条件1，就执行语句1，然后退出整个if；如果不满足条件1，则判断是否满足条件2……

    1. 只有一个语句能执行。

    2. else if里的条件理论上可以任意多个。

    三元表达式¶

    有三元运算符 ？和 ：组成的式子为三元表达式。

    语法结构：

    '条件表达式'? '表达式1':'表达式2'

    执行思路：

    如果条件表达式结果为真，返回式1的值；如果条件表达式结果为假，返回式2的值。

    var result = ptt > 12 ? '很强':'有点菜'; // 等价于if语句
console.log(result);
alert(result);

    switch语句¶

    当要针对变量设置一系列特定值的选项时，就可以使用switch语句。

    语法结构：

    switch('表达式'){
    case value1:
         执行语句1；
         break； 
    case value2:
         执行语句2；
         break；
    default:
         执行最后的语句; 
}

    执行思路：把表达式的值与case后面的value进行匹配，如果匹配上了，就执行对应的语句，然后退出switch（每一个case最后一定要加上break）如果都没有匹配上，就执行default里的语句。

    注意：匹配时必须是全等，值和数据类型都一致

    switch (2) {
    case 1:
        console.log('这是1');
        break;
    case 2:
        console.log('这是2');
        break;
    default:
        console.log('啥也不是');
}

    循环结构¶

    使某些代码重复执行，完成规律性的重复操作。

    一组被重复执行的语句称为循环体，而能否继续重复，取决于循环的终止条件。

    for循环语句¶

    语法结构：

    for(初始化变量 ; 条件表达式 ; 操作表达式){
    // 循环体
}

    • 初始化变量：一个普通变量，通常作为计数器使用。

    • 条件表达式：用来决定每一次循环是否执行。

    • 操作表达式：每次循环最后执行的代码，经常用于计数器变量（递增或递减）

    执行思路（顺序）：初始化变量→条件表达式→循环体→操作表达式。

    例：

    const prompt = require('prompt-sync')();
var num = prompt('请输入人数：');
var sum = 0;
var ave = 0;
for (var i = 1; i <= num; i++) {
    var score = parseInt(prompt('请输入第' + i + '个人的分数'));
    // 注意一定要转换类型，因为从prompt获取的值是字符串。
    sum = sum + score;
}
ave = parseInt(sum / num);
console.log('平均分数是' + ave);

    断点调试¶

    在程序的某一行设置一个断点（vscode里直接单击行号前面的小红点，浏览器按F12进入source以后操作）调试时，程序会在断点处停止。

    在浏览器的调试功能里按F11还可以让程序一步一步执行，并且可以点击watch，观察变量的值的变化。

    双重for循环¶

    语法结构：

    for(初始化变量 ; 条件表达式 ; 操作表达式){ //外层
    for(初始化变量 ; 条件表达式 ; 操作表达式){ //里层
       // 循环体
    }
}

    执行思路：可以把里面的循环看作是外面循环的语句，外层执行一次，里层执行全部

    例：

    for (var i = 1; i <= 3; i++){
    console.log('这是外层，第' + i + '次');
    for (var j = 1; j <= 3; j++){
        console.log(' - 这是内层，第' + j + '次');
    }
}

    输出结果：

    这是外层，第1次
 - 这是内层，第1次
 - 这是内层，第2次
 - 这是内层，第3次
这是外层，第2次
 - 这是内层，第1次
 - 这是内层，第2次
 - 这是内层，第3次
这是外层，第3次
 - 这是内层，第1次
 - 这是内层，第2次
 - 这是内层，第3次

    tips：分析比写代码更重要，如果核心算法想不到的话，就先分析它的执行过程。

    while语句¶

    语法结构：

    while (条件表达式){
    // 循环体
}

    执行思路：当条件表达式结果为true时，则执行循环体，否则退出循环。

    var num = 1;
while (num <= 100){
    console.log('hello');
    num++;
}

    和for循环语句一样需要有计数器和操作表达式，防止变成无限循环把电脑卡死。

    do while 语句¶

    语法结构：

    do {
    // 循环体
} while (条件表达式)

    执行思路：与while语句不同之处在于：先执行一遍再判断条件。

    (也就是说，无论如何都会至少执行一次循环体代码。)

    var i = 1;
do {
    console.log('hello');
    i++;
} while (i <= 100)

    冒泡排序算法¶

    重复访问要排序的数组，一次比较2个元素，如果顺序错误，就交换这两个元素，重复直到没有再需要交换为止。

    用双层for循环，外层决定访问元素的次数，内层决定要交换元素的次数。

    var arr = [5, 4, 3, 2, 1];
for (var i = 0; i <= arr.length - 1; i++) {
    for (var j = 0; j <= arr.length - 1 - i; j++){
        if (arr[j] > arr[j + 1]) {
            var temp = arr[j];
            arr[j] = arr[j + 1];
            arr[j + 1] = temp;
        }
    }
}
console.log(arr);

getSum(1237, 2931);

getSum1(1237, 2931);

    函数的返回值 return¶

    • return语句：把函数的值返回给调用者 return 需要返回的结果;

    例：利用函数求一个数组中的最大值

    function getArrMax(arr) {
    var max = arr[0];
    for (var i = 1; i < arr.length; i++) {
        if (arr[i] > max) {
            max = arr[i]
        }
    }
    return max;
}
console.log(getArrMax([1, 34, 6, 2, 46, 52])); 
// 52

    实际开发中，经常用一个变量来接受函数的返回结果，使用更简单。如果没有return，这个函数的值就会显示为Undefined。

    var re = getArrMax([1, 34, 6, 2, 46, 52]);
console.log(re);

    return会终止函数，并且只能返回一个值（但也可以是一个数组）

    arguments¶

    当我们不确定有多少个参数的时候，可以用arguments来获取。arguments实际上是所有函数都有的一个内置对象（也只有函数才有这个对象），存储了所有传递过来的实参。

    arguments的展示形式是伪数组，具有以下特点：

    • 具有length属性

    • 按索引存储数据

    • 不具有push、pop等方法

    例：找出任意个数的最大值。

    function getMax() {
    var max = arguments[0];
    for (var i = 1; i < arguments.length; i++)
        //因为已经把max设为参数的第一个了，所以遍历的时候从第二个开始。
        if (arguments[i] >= max) {
            max = arguments[i];
        }
    return max;
}

console.log(getMax(4,54,667,293,375,29)); // 调用

    例：用函数翻转数组

    function reverse(arr) {
    var newArr = [];
    for (var i = arr.length - 1; i >= 0; i--){
        // 这里是从倒数第一个值开始往前走，所以用 i--；
        newArr[newArr.length] = arr[i];
    }
    return newArr;
}

var arr1 = reverse([1, 3, 5, 7, 9]);
console.log(arr1);

    函数的另一种声明方式：表达式法¶

    声明时：var 变量名 = function ( ){ 需要封装的代码 };

    调用时：变量名( );

    这种声明方式和一般声明变量的方式差不多。函数表达式也可以传递参数。

    作用域¶

    指的是代码名（变量）起作用的范围，分为全局作用域和局部作用域。

    • 全局作用域：整个Script标签内，或者一个单独的js文件内。

    • 局部作用域：只在函数内部起作用，也叫函数作用域。

    根据作用域不同，可以把变量分为全局变量和局部变量。

    • 注意：如果在函数内部，没有声明，直接赋值，该变量会被视为全局变量。

    let和const（来自AI）¶

    原本的课程里没有，所以在这里补充。

    let 和 const 是 ES6（2015年）新增的变量声明方式，用于解决传统 var 存在的作用域混乱、变量提升等问题，是现代 JavaScript 开发的基础。下面我们来详细对比它们的特性和使用场景：

    let：用于声明可修改的变量¶

    • 块级作用域：只在 { } 包裹的代码块（如 if、for、函数体等）内有效，外部无法访问。

    javascript if (true) { let a = 10; console.log(a); // 10（块内可访问） } console.log(a); // 报错：a is not defined（块外不可访问）

    • 不存在变量提升：必须先声明后使用，否则会报错（var 会提升到作用域顶部，值为 undefined）。

    javascript console.log(b); // 报错：Cannot access 'b' before initialization let b = 20;

    • 不允许重复声明：同一作用域内不能重复声明同名变量（包括 var 声明的变量）。

    javascript let c = 30; let c = 40; // 报错：Identifier 'c' has already been declared var d = 50; let d = 60; // 同样报错

    • 暂存死区（TDZ）：在变量声明前的区域，变量处于“暂存死区”，无法被访问（避免了 var 的意外赋值问题）。

    const：用于声明不可修改的常量¶

    • 声明时必须赋值：否则会报错（let 可以先声明后赋值）。

    javascript const e; // 报错：Missing initializer in const declaration const e = 70; // 正确

    • 块级作用域：和 let 一样，只在当前代码块内有效。

    javascript { const f = 80; console.log(f); // 80 } console.log(f); // 报错：f is not defined

    • 不允许重复声明：规则和 let 一致。

    • 值不可修改？（注意！）：

    对于 基本数据类型（数字、字符串、布尔值等）：值确实不可修改，强行修改会报错。

    javascript const g = 90; g = 100; // 报错：Assignment to constant variable

    对于 引用数据类型（对象、数组等）：变量存储的是“引用地址”，地址不可改，但对象/数组的 内部属性可以修改。

    javascript const obj = { name: "张三" }; obj.name = "李四"; // 允许：修改对象内部属性 console.log(obj); // { name: "李四" } obj = { age: 20 }; // 报错：试图修改引用地址

    let、const 与 var 的对比¶

    特性 var let const
    作用域 函数级/全局 块级 块级
    变量提升 有（提升到作用域顶） 无（暂存死区） 无（暂存死区）
    重复声明 允许 不允许 不允许
    初始赋值 可选 可选 必须
    值可修改 允许 允许 基本类型不可改，引用类型内部可改

    • 优先使用 const：如果变量的值不需要修改（如配置项、固定常量），用 const 可以避免意外修改，代码更安全。

    • 其次使用 let：当变量需要被重新赋值时（如循环计数器、条件判断的临时变量），再用 let。

    • 避免使用 var：var 的作用域和提升特性容易导致 bugs（如循环中闭包问题），现代开发中已基本被 let/const 替代。 示例：

    javascript // 推荐写法 const PI = 3.14159; // 常量用 const let count = 0; // 会被修改的变量用 let for (let i = 0; i < 5; i++) { // 循环变量用 let（块级作用域，每次循环都是新变量） count += i; }

    掌握 let 和 const 是理解 ES6 及后续版本的基础，也是现代前端框架（如 Vue、React）开发中的必备知识。

    预解析¶

    • JS运行时分为两步：预解析、代码执行。

    • 预解析时，JS引擎会把代码里所有的var和function都提升到当前作用域的最前面。

    • 执行时按照代码书写的顺序从上到下执行。

    • 变量预解析/变量提升：把所有变量声明提升到作用域最上面（不提升赋值操作）

    • 函数预解析/提升：把所有函数声明提升到作用域最上面（不调用函数）

    箭头函数 => (来自AI)¶

    箭头函数（=>）是 ES6 新增的函数语法，与普通函数（function 声明）在语法和行为上有显著区别，核心差异体现在 this 指向、语法简洁性、使用场景 等方面。下面详细对比：

    语法差异¶

    普通函数¶

    • 必须用 function 关键字声明，可包含完整的函数体（{ } 和 return）

    javascript // 函数声明 function add(a, b) { return a + b; } // 函数表达式 const multiply = function(a, b) { return a * b; };sd

    箭头函数¶

    • 用 => 替代 function 关键字，语法更简洁

    • 单参数可省略括号 ()，单语句返回可省略 { } 和 return（隐式返回）

    javascript // 基础形式（多参数） const add = (a, b) => { return a + b; }; // 简化：单语句返回可省略 { } 和 return const add = (a, b) => a + b; // 简化：单参数可省略 () const double = num => num * 2; // 特殊：返回对象时需用 () 包裹（避免 { } 被解析为代码块） const getUser = () => ({ name: "张三", age: 20 });

    核心差异：this 指向¶

    这是箭头函数与普通函数最关键的区别，直接影响它们的使用场景。

    普通函数¶

    • this 指向 调用时的上下文对象，取决于“谁调用它”：

    • 全局调用时，this 指向全局对象（浏览器中是 window，Node.js 中是 global）

    • 对象方法调用时，this 指向该对象

    • 事件绑定中，this 指向触发事件的元素

    • 用 call/apply/bind 可修改 this 指向

      javascript const obj = { name: "张三", sayHi: function() { console.log(this.name); // "张三"（this 指向 obj） } }; obj.sayHi(); // 全局调用时 this 指向 window const globalFunc = obj.sayHi; globalFunc(); // undefined（浏览器中 window.name 默认为空）

    箭头函数¶

    • 没有自己的 this，this 继承自 声明时的父级作用域（“词法作用域 this”）

    • 无法通过 call/apply/bind 修改 this 指向（参数会被忽略）

    javascript const obj = { name: "张三", sayHi: () => { console.log(this.name); // undefined（this 继承自父级，即全局 window） }, delaySayHi: function() { // 普通函数中的 this 指向 obj setTimeout(() => { // 箭头函数的 this 继承自 delaySayHi 的 this（即 obj） console.log(this.name); // "张三" }, 1000); } }; obj.sayHi(); // undefined obj.delaySayHi(); // 1秒后输出 "张三"

    经典场景：箭头函数常用于回调函数（如 setTimeout、数组方法），避免普通函数中 this 指向混乱的问题。

    其他差异¶

    特性 普通函数 箭头函数
    arguments 对象 有（包含所有参数的类数组） 无（需用剩余参数 ...args）
    构造函数 可作为构造函数（new Func()） 不可（会报错）
    原型 prototype 有原型属性 无原型属性
    Generator 函数 可使用 function* 声明 不可（不支持 yield）

    示例：arguments 与剩余参数¶

    // 普通函数：用 arguments 获取参数
function sum() {
let total = 0;
for (let i = 0; i < arguments.length; i++) {
total += arguments[i];
}
return total;
}
sum(1, 2, 3); // 6
// 箭头函数：用 ...args 剩余参数替代 arguments
const sum = (...args) => {
return args.reduce((total, num) => total + num, 0);
};
sum(1, 2, 3); // 6

    使用场景建议¶

    优先用箭头函数的场景¶

    • 回调函数：如 setTimeout、map、filter 等，避免 this 指向问题

    javascript const numbers = [1, 2, 3]; const doubled = numbers.map(num => num * 2); // 简洁且无 this 困扰

    • 简短逻辑函数：单语句返回的简单函数（如工具函数）

    • 需要继承父级 this 的场景：如对象方法中的嵌套函数（见上文 delaySayHi 示例）

    优先用普通函数的场景¶

    • 对象方法：因为箭头函数的 this 不指向当前对象

    javascript const obj = { value: 10, getValue: function() { // 必须用普通函数 return this.value; } };

    • 构造函数：需要通过 new 创建实例时

    javascript function User(name) { this.name = name; // 普通函数可作为构造函数 } const user = new User("张三");

    • 需要 arguments 对象或 prototype 时

    总结¶

    箭头函数的设计初衷是 简化语法 并 解决普通函数中 this 指向混乱 的问题，但它并非普通函数的替代品。实际开发中需根据场景选择：需要动态 this 时用普通函数，需要简洁语法或固定 this 时用箭头函数。理解 this 指向的差异是掌握两者的关键。

    倒计时¶

    • 思路：输入的时间减去现在的时间就是倒计时

    • 用时间戳来做，得到毫秒数倒计时

    • 把毫秒转换为天数、时、分、秒

    function countDown(time) {
    const now = +new Date();
    const input = +new Date(time);
    const times = (input - now) / 1000;
    const d = parseInt(times / 60 / 60 / 24);
    const h = parseInt(times / 60 / 60 % 24);
    const m = parseInt(times / 60 % 60);
    const s = parseInt(times % 60);
    return d + '天' + h + '小时' + m + '分钟' + s + '秒';
}
console.log(countDown('2028-3-1 18:00'));

    数据类型¶

    简单类型与复杂类型¶

    • 简单类型（值类型）：在存储时存储的是值本身，例如string、number、boolean、undefined

    • 复杂类型（引用类型）：在存储时只是存储了地址；可以通过new关键字新建，例如object、array、date

    堆和栈（数据结构中的概念）¶

    • 栈：由操作系统自动分配释放存放函数的参数值、局部变量的值等。简单数据类型存放到栈中。

    • 堆：存储复杂数据类型，由程序员分配释放，若程序员不释放，则由垃圾回收机制回收。复杂数据类型存放到堆中。

    • 复杂类型变量的栈空间存放的是地址，而真正的对象存放在堆空间中。

    API¶

    全称为Application Programming Interface，应用程序编程接口，是一些预定义的函数，作用是提供了基于某软件或硬件不需要访问源码就能实现某种功能的能力。

    Web API是浏览器提供的一套API，包括操作浏览器的BOM和操作页面元素的DOM

    DOM¶

    文档对象模型(Document Object Model)，是HTML或XML的标准编程接口。

    W3C已经定义了一系列DOM接口，通过它们可以改变网页的内容、结构、样式。

    • 文档：用document表示。一个页面就是一个文档。

    • 元素：用element表示。页面中所有标签都是元素。

    • 节点：用node表示。网页中所有内容都是节点（包括标签、属性、文本、注释）

    • DOM把以上内容都视为对象。

    获取元素¶

    有以下几种方式：

    • 根据ID获取

    • 根据标签名获取

    • 通过HTML5新方法获取

    • 特殊元素获取

    根据ID获取元素¶

    document.getElementByID() 返回一个匹配特定ID的元素对象

    如果在当前Document下没有找到，则会返回null。

    注意：Script必须写在元素的下面、参数必须是字符串（加引号）

    • 用console.log() 会返回整个标签。

    • 用console.dir() 会返回元素对象名、属性和方法。

    根据标签名获取元素¶

    • document.getElementsByTagName()返回指定标签的元素对象的集合（数组）

    用console.log() 会返回数组，即使页面中没有要找的标签，也会返回一个空的数组。

    • element.getElementsByTagName() 返回某个特定元素内部所有指定标签的子元素

    根据类名获取元素（现在最常用这个）¶

    • getElementsByClassName() 根据类名返回元素对象集合

    • querySelector('选择器') 根据指定选择器返回第一个元素对象（选择器的格式同CSS）

    • querySelectorAll() 返回指定选择器的所有元素对象集合

    获取特殊元素body和html¶

    • document.body 获取body

    • document.documentElement 获取html

    事件¶

    事件可以简单理解为”触发——响应机制“

    三要素：事件源、事件类型、事件处理程序

    • 事件源：事件被触发的对象（谁）

    • 事件类型：如何触发（什么操作）

    • 事件处理程序：通过一个函数赋值的方式完成

    // 1.获取事件源
const div = document.querySelector('div');
// 2.绑定事件，添加处理程序
div.onclick = function(){
    console.log('我被选中了');
}

    操作元素¶

    1.修改元素内容¶

    • element.innerText 获取元素内容，但去除html标签

    • element.innerHTML 获取元素内容，包括标签

    • 这两个属性都是可读写的。

    例子：

    <div></div> <!--空盒子-->

    innerText：不识别标签且不保留空格和换行

    const div = document.querySelecter('div');
div.innerText='<strong> DABC </strong> ABC';
// 这时候会发现<strong>这个标签没有识别到，被当成文本的一部分了
// 而且strong标签也没有生效（文字没有加粗）

    innerHTML：识别标签，且保留空格和换行

    const div = document.querySelecter('div');
div.innerHTML='<strong> DABC </strong> ABC';
// 这样strong标签会被识别

    例如src、href，再例如ID、title……

    操作的过程同上

    3.表单元素操作¶

    例如type、value、checked、selected、disabled……

    表单中的文字内容无法通过inner修改，但可以用value
